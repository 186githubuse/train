**Gathered:** 2026-06-29

<domain>
## Task Boundary

实现后台数据库 V1 第二阶段：学生注册/登录接口。基于现有 `server/` 骨架和 `schema.sql`，新增学生账号数据访问层、密码哈希、`POST /api/auth/student/register`、`POST /api/auth/student/login`、`GET /api/auth/me`。本阶段仍不接前端、不做学习数据同步接口、不部署生产。

</domain>

<decisions>
## Implementation Decisions

### 学生登录账号字段
- 选择 **手机号 + 密码**。注册接口应接收 `phone`、`password`、`name`、`grade` 等字段；手机号作为学生唯一登录标识。
- 不接短信验证码；手机号只是账号字段，后续如需短信验证另起阶段。
- 避免使用姓名+年级作为登录标识，防止重名与数据归属混乱。

### 密码存储方案
- 选择 **bcryptjs**。新增后端依赖 `bcryptjs`，密码只保存 hash，不保存明文。
- 建议在代码中集中封装 hash/verify，避免控制器里散落 bcrypt 调用。

### 验证方式
- 本阶段采用 **无 MySQL 骨架验证**：通过 `npm install`、`node --check`、路由/模型文件结构检查、关键关键词检查验证实现完整性。
- 不要求本地创建真实 MySQL 数据库，不要求本阶段 curl 注册/登录真实落库；真实 MySQL 联调放到后续“数据库准备/接口联调”阶段。

### Claude's Discretion
- 保持现有约束：不接前端、不做同步接口、不部署生产。
- 接口返回保持现有统一 JSON 风格：`{ code, message, data }`。
- `GET /api/auth/me` 使用第一阶段已有 JWT middleware 保护。

</decisions>

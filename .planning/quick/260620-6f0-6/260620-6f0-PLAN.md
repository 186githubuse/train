# Quick Task 260620-6f0: 新增模块首页(杨红作文6大模块壳)最小可用版 - Plan

**Created:** 2026-06-19
**Mode:** quick
**Status:** Ready for execution

## Goal

在现有 app 最外层新增一个「模块首页」视图，登录后先进入该页，展示杨红作文 6 大模块壳：感觉训练为可进入的大卡，其余模块为锁定占位。右屏专题选择页改造不在本次范围。

## Scope

### In scope
- 新增 `views/moduleHome.js`
- 新增 `css/views/moduleHome.css`
- `js/router.js` 注册 `moduleHome`，让该视图自渲染 header
- `js/main.js` 老用户初始落地页改为 `moduleHome`（可回退到 `trainingCamp`）
- `views/onboarding.js` 注册/登录完成后进入 `moduleHome`
- `index.html` 引入 moduleHome CSS，带版本号
- 基础本地验证：静态服务器可加载，浏览器控制台无明显模块加载错误（或 curl 检查资源存在）

### Out of scope
- `topicDetail` 面包屑 / 去前缀 / 配图改造
- 底部导航结构大改
- 数据库、后端、AI 评分、专题数据改动

## Implementation Tasks

### Task 1 — Add `moduleHome` view
**Files:** `views/moduleHome.js`

**Action:**
- 创建 `renderModuleHome()`，渲染：
  - `#app-header`：模块首页标题 / 副标题 / 小状态胶囊
  - `#app-content`：状态栏 + 6 模块卡片
- `感觉训练` 大卡点击 `window.__router.navigate('trainingCamp')`
- 锁定模块点击 `window.__showToast('敬请期待')`
- 数据内聚在视图文件中，避免引入新全局数据文件

**Verify:**
- 动态 import 可加载
- 点击可进入 trainingCamp
- 锁定模块弹 toast

### Task 2 — Add moduleHome CSS
**Files:** `css/views/moduleHome.css`, `index.html`

**Action:**
- 从 `mockup-layout.html` 左屏提炼样式，命名统一加 `mh-` 前缀，避免污染现有 `.mod` / `.statusbar`
- 移动端 1 列；桌面 ≥1024px：3 列网格，感觉训练大卡跨两行，36讲横条跨两列
- 引入 CSS：`css/views/moduleHome.css?v=20260620a`

**Verify:**
- 页面布局在移动/桌面都不破坏现有 app shell
- CSS 不影响 trainingCamp/topic 等现有类名

### Task 3 — Wire routing and landing page
**Files:** `js/router.js`, `js/main.js`, `views/onboarding.js`

**Action:**
- `VIEW_MAP` 增加 `moduleHome`
- `viewsWithHeader` 增加 `moduleHome`
- `goBack()` 空历史回退到 `moduleHome`
- `main.js` 老用户默认从 `trainingCamp` 改为 `moduleHome`，旁边注释说明回退方法
- `onboarding.js` `_enterApp()` 从 `trainingCamp` 改为 `moduleHome`，旁边注释说明回退方法

**Verify:**
- 新用户完成注册 / 老用户刷新都会进模块首页
- 底部导航仍能进入训练营
- 从训练营内返回行为合理

## Rollback
